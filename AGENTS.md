# AGENTS.md

## 목적
이 저장소의 JavaScript 앱을 안전하게 분석하고, 기존 동작을 최대한 유지하며 개선한다.

## 작업 원칙
- 먼저 구조를 읽고 엔트리포인트와 상태 흐름을 파악한 뒤 수정한다.
- 한 번에 큰 변경을 하지 말고 작은 diff로 나눈다.
- 기존 동작을 바꾸는 경우 반드시 이유를 설명한다.
- 추정하지 말고 실제 파일/함수 기준으로 설명한다.
- 불필요한 의존성 추가는 피한다.

## 코드 변경 원칙
- 먼저 분석 → 변경 제안 → 승인 후 수정
- 가능하면 순수 JS/HTML/CSS 구조를 유지
- 리팩토링 시 이름만 바꾸지 말고 책임 분리를 분명히 할 것
- dead code가 확인되면 삭제 후보로 표시

## 확인할 것
- 엔트리포인트 파일
- 전역 상태
- 이벤트 바인딩 위치
- 렌더링 흐름
- 잠재 버그와 중복 로직

## 출력 형식
- 먼저 현재 구조 요약
- 그 다음 문제점
- 그 다음 개선 우선순위
- 코드 수정은 내가 요청할 때만 제안
